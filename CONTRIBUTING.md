# Contributing

Hi — thanks for taking an interest in this project. It's a personal project tied to my YouTube channel, [@groundupcoder](https://www.youtube.com/@groundupcoder), and maintained by one person in spare time. A few notes on how to engage.

## Bug reports and questions

Please open an issue. Bug reports, design questions, things in the code that confused you while reading — all welcome. These are genuinely useful and I appreciate them.

## Pull requests

I do accept code contributions, but I'm cautious about them, for two reasons:

- The codebase is partly an artifact of me figuring things out, and I'd like to make most of the design decisions myself rather than have them made for me.
- I'd rather build a working relationship with regular contributors than merge code from people I haven't talked to.

So:

- **For non-trivial changes, please open an issue first to discuss.** Tell me what you want to change and why; I'll respond with thoughts. This avoids you spending time on a change I might not want.
- **I may close a PR even if the change is correct,** if I'd rather make the fix myself in my own style. If I do, I'll credit your diagnosis in the resulting commit.

If that sounds reasonable, please go ahead.
